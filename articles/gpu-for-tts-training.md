## 結論(先に)

TTS(音声合成)モデルの学習用GPUは、カタログの**ピークTFLOPS(演算性能)ではなく、「メモリ帯域」と「VRAM容量」で選ぶ**のが正解です。

なぜなら、TTSモデルは画像・言語モデルに比べて小さく、**学習がメモリ帯域律速(memory-bandwidth-bound)になりやすい**からです。この特性を知らないと、「AI向け」を謳う高価な製品を買ったのに大して速くならない、という失敗をします。

:::message
本記事の数値は2026年年央時点のもので、実売価格は変動します。ベンチはCPU/GPU1枚・単精度混在の実測で、環境により差が出ます。
:::

## TTS学習のボトルネックはどこにあるか

GPU学習のボトルネックは、大きく5つに分けられます。

1. **データローダ(CPU)** … 音声読み込み・リサンプル・メル計算・g2p
2. **PCIe転送** … CPU→GPUへバッチを送る帯域
3. **GPU演算** … 行列積・畳み込み(Tensor Core)
4. **GPUメモリ帯域** … VRAMへの読み書き速度
5. **VRAM容量** … モデル+バッチが載るか

TTSモデル(VITS、各種ボコーダなど)は、**巨大な行列積が少なく、小さな畳み込み・GRU・STFT/iSTFT・正規化・活性化といった「軽い演算の集まり」**でできています。これらは演算量より**メモリアクセス量**が支配的、つまり **④のメモリ帯域が律速**になりがちです。

その結果、**ピーク演算性能(TFLOPS)を上げても効きにくく、メモリ帯域を上げると素直に速くなる**、という傾向が生まれます。

## 自分の学習がどれ律速か、まず測る

選ぶ前に、**手元の学習が何律速なのかを測る**のが一番確実です。学習を回しながら次を実行します。

```bash
nvidia-smi dmon -s puct -c 10
```

出てくる列のうち、**`sm`(演算ユニット利用率)と `mem`(メモリコントローラ利用率)** を見ます。判定は次のとおりです。

| 観測パターン | 律速要因 | 効くGPU強化 |
|---|---|---|
| `mem` が100%張り付き / `sm` は中途半端(70〜85%) | **メモリ帯域律速** | 帯域の広いGPU |
| `sm` も `mem` も低く、CPUが忙しい | **データローダ律速** | GPU換えても無駄。CPU/前処理改善 |
| `sm` が100%張り付き / 大きなmatmul主体 | **演算律速** | TFLOPSの高いGPU |
| `rxpci`(PCIe受信)が高い | **PCIe転送律速** | 統合メモリ or x16化 |

### 実例

筆者の環境(ノート用 RTX 3070、16GB)で軽量な音響モデルを学習しながら測ると、次のようになりました。

```
sm    mem   pclk   pwr
78%   100%  1875   88W   ← memが100%張り付き、smは78%、電力は上限115Wに未達
```

**`mem` が100%で `sm` が余っている(78%)、しかも消費電力が上限に届いていない** ——これは典型的な**メモリ帯域律速**の波形です。「演算ユニットが帯域待ちで遊んでいる」状態なので、**演算性能をいくら足しても速くなりません**。効くのは帯域だけです。

:::message
`--cache-mels` のようにメルを事前計算してRAMに載せていると、データローダ律速は避けられます。その場合ボトルネックはGPU内部(帯域 or 演算)に移ります。
:::

## GPUスペックの読み方

TTS学習の観点で、見るべき順に並べると:

1. **メモリ帯域(GB/s)** … 前述のとおり最重要になりやすい
2. **VRAM容量(GB)** … モデル+バッチが載るか。足りないとOOMするかバッチを削るしかない
3. **演算性能(BF16/FP16 TFLOPS)** … 大きなモデルや演算律速になったとき効く

そして落とし穴が2つあります。

- **「Laptop」版は同名でも別物**。ノート用は帯域・電力が低く、熱でクロックが落ちて(サーマルスロットリング)公称性能を出し切れません。**据置のデスクトップの方が有利**です。
- **VRAMは「今の使用量」でなく「これからやる最大構成」で選ぶ**。小さいモデルは数GBしか使いませんが、モデルを大きくした瞬間に足りなくなります。

## GPU候補の比較(2026年年央)

TTS学習の観点で主要GPUを並べます。帯域は「ノート用3070(≒448 GB/s)」を基準にした倍率つきです。

| GPU | VRAM | メモリ帯域 | 3070Laptop比 | 実売の目安 | 一言 |
|---|---|---|---|---|---|
| RTX 3070 Laptop(基準) | 16GB | ~448 GB/s | 1.0× | — | ノート。throttleあり |
| **NVIDIA GB10 / DGX Spark** | 128GB | **273 GB/s** | **0.6×** | 約50万〜 | 容量は別格だが**帯域は基準未満** |
| 中古 RTX 3090 | 24GB | 936 GB/s | 2.1× | 約9〜13万 | ML定番の激安枠 |
| 中古 RTX 4090 | 24GB | 1008 GB/s | 2.3× | 約20万〜 | 演算も強い |
| 新品 RTX 5070 | 12GB | 672 GB/s | 1.5× | 約9万 | 安いが**VRAMは12GB** |
| 新品 RTX 5070 Ti | 16GB | 896 GB/s | 2.0× | 約15万 | 帯域2倍・16GB。バランス良 |
| 新品 RTX 5080 | 16GB | 960 GB/s | 2.1× | 約20万〜 | GDDR7 |
| 新品 RTX 5090 | 32GB | **1792 GB/s** | **4.0×** | 約40万〜 | 帯域も容量も別格 |

## 「AI向け」GB10の落とし穴

DGX Spark に載る **NVIDIA GB10** は「128GB統合メモリ」「FP4で1 PetaFLOP」と派手ですが、**TTS学習には要注意**です。

- **メモリ帯域が 273 GB/s しかない** … これは上表のとおり、ゲーミングGPUどころか**ノート用3070(448 GB/s)より低い**。帯域律速なTTS学習では、買っても律速が改善しません。
- **「1 PFLOP FP4」は推論向けの数字** … 学習は BF16/FP16/FP32 で行うため、FP4の性能は効きません。GB10のBF16実測は約100 TFLOPSで、これは強力ですが、**帯域律速の学習ではその演算力を活かせません**。
- **統合メモリ・128GB容量が効くのは、「PCIe転送律速」や「超大バッチ・大モデル」のとき**。小〜中規模のTTS学習ではメリットが刺さりにくい。

つまり **「メモリ容量は欲しいが、帯域律速な小さめモデルを回す」なら、GB10より普通のゲーミングGPUの方が速くて安い**、という逆転が起きます。

## 予算別おすすめ

律速が**帯域**である前提(多くのTTS学習が該当)での指針です。

- **〜10万円** … **中古 RTX 3090(24GB/936 GB/s)** が最強コスパ。新品にこだわるなら **RTX 5070(12GB/672 GB/s)**。ただし5070は12GBなので、大きめモデルを見据えるなら3090の24GBが安心。
- **〜15万円** … **新品 RTX 5070 Ti(16GB/896 GB/s)**。帯域2倍・16GBで新品保証。あるいは**中古 RTX 4090(24GB)**。
- **30万円以上** … **RTX 5090(32GB/1792 GB/s)**。帯域4倍・大容量で、大きいモデルまで長く戦える。
- **物理GPUを持ちたくない** … RunPod / Vast.ai 等の**クラウドで時間借り**。初期投資ゼロ。

## 購入前チェックリスト

- [ ] **VRAMは「これからやる最大モデル」基準**で選んだか(今の使用量ではない)
- [ ] **Laptopではなくデスクトップ**か(据置=throttleしない)
- [ ] メモリ帯域が今の環境より十分広いか
- [ ] 電源容量は足りるか(RTX 5070 Ti=300W、3090/4090=350W → 750W以上推奨)
- [ ] 中古は**元マイニング品・サーマルパッド劣化**に注意(保証つきショップ推奨)

## まとめ

- TTS学習は**メモリ帯域律速になりやすい** → GPUは**帯域とVRAM**で選ぶ
- 買う前に **`nvidia-smi dmon` で `sm` と `mem` を見て**、自分の学習が何律速かを測る
- **同名のLaptop版は別物**、据置デスクトップが有利
- 「AI向け」の**GB10は帯域が意外と低く**、帯域律速なTTS学習では割に合わないことがある
- 迷ったら**中古3090(コスパ)/ 新品5070 Ti(新品バランス)/ 5090(全部盛り)** から予算で選ぶ

「ピーク性能」ではなく「自分の学習の律速」で選ぶ——これがTTS学習用GPU選びの肝です。
